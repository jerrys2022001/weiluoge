Bluetooth GATT Attribute Reference

Characteristic Declaration (0x2803)

org.bluetooth.attribute.gatt.characteristic_declaration

Characteristic Declaration defines how a characteristic is announced in the GATT attribute table. It combines a one-byte properties bitfield, a two-byte characteristic value handle, and a characteristic UUID (gatt_uuid) so clients can discover capabilities before reading or writing values.

Quick Summary

UUID 0x2803 declares each characteristic entry in a GATT service. The payload is 5 bytes for 16-bit UUID characteristics or 19 bytes for 128-bit UUID characteristics. Byte 0 is properties, bytes 1-2 are value handle (little-endian), and the remaining bytes are characteristic UUID.

Field Specification

All declaration fields are mandatory and parsed in strict order.

Field Requirement Format Typical Size Description
Characteristic Properties Mandatory uint8 1 byte Bitmask describing capabilities such as Read, Write, Notify, and Indicate.
Characteristic Value Handle Mandatory uint16 2 bytes Handle of the characteristic value attribute to read/write/subscribe against.
Characteristic UUID Mandatory gatt_uuid 2 or 16 bytes 16-bit or 128-bit UUID identifying the characteristic type.

Byte Layout and Decode

Validate payload length first: 5 bytes for 16-bit UUID declarations or 19 bytes for 128-bit UUID declarations. 

function decodeCharacteristicDeclaration(payload) {
  if (payload.length !== 5 && payload.length !== 19) {
    throw new Error("Invalid 0x2803 payload length");
  }

  const properties = payload[0];
  const valueHandle = payload[1] | (payload[2] << 8);
  const uuidBytes = payload.slice(3);

  const is16BitUuid = uuidBytes.length === 2;
  const characteristicUuid = is16BitUuid
    ? `0x${(uuidBytes[0] | (uuidBytes[1] << 8)).toString(16).padStart(4, "0")}`
    : Array.from(uuidBytes).map((b) => b.toString(16).padStart(2, "0")).join("");

  return { properties, valueHandle, characteristicUuid };
}

// Example 16-bit declaration payload: 12 2A 00 A0 2A
// properties = 0x12 (Read + Notify)
// valueHandle = 0x002A
// characteristicUuid = 0x2AA0

Use Cases and Integration Notes

Service Discovery Parsers

Decode declarations to build client-side capability maps before issuing reads, writes, or subscriptions.

Auto-generated BLE SDKs

Use properties bits to enforce operation rules and avoid unsupported ATT operations in generated APIs.

Debug and Compliance Tools

Verify declaration consistency against actual handles and UUIDs to catch firmware table mistakes early.

Implementation Checklist

  • Decode properties as bit flags and map to capability booleans for app logic.
  • Treat declaration handle and characteristic value handle as separate addresses.
  • Support both 16-bit and 128-bit characteristic UUID payload variants.
  • Cross-check discovered declaration UUID against expected profile schema in QA.

FAQ

What is GATT Characteristic Declaration 0x2803?

It is the metadata attribute that declares characteristic properties, the characteristic value handle, and characteristic UUID.

How long is the declaration payload?

5 bytes with 16-bit UUIDs or 19 bytes with 128-bit UUIDs.

How do I detect Notify support?

Check bit 4 of Characteristic Properties. If set, the characteristic supports notifications.

What does value handle point to?

It points to the separate characteristic value attribute, which is where data read/write operations occur.

Should I parse declaration before enabling CCCD?

Yes. Read declaration properties first to confirm Notify/Indicate support before writing CCCD.
