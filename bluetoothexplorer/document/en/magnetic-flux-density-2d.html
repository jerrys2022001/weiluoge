Magnetic Flux Density 2D (0x2AA0)

org.bluetooth.characteristic.magnetic_flux_density_2D

This characteristic represents two-dimensional magnetic flux density with two mandatory sint16 values: X-Axis and Y-Axis. Both fields use gauss as the unit and a decimal exponent of 0.

Quick Summary

Characteristic 0x2AA0 reports 2D magnetic flux density as a fixed-length payload. Parse bytes [0..1] as X-axis and bytes [2..3] as Y-axis, both signed 16-bit little-endian integers in gauss.

Byte Layout and Decode

Payload size is always 4 bytes. Decode each axis as little-endian signed 16-bit data. 

function decodeMagneticFluxDensity2D(dataView, offset = 0) {
  const xAxisGauss = dataView.getInt16(offset, true);
  const yAxisGauss = dataView.getInt16(offset + 2, true);
  return { xAxisGauss, yAxisGauss };
}

// Example payload: 34 12 CE FF
// X = 0x1234 = 4660 gauss
// Y = 0xFFCE = -50 gauss
// Conversion: microtesla = gauss * 100

Implementation Checklist

  • Validate UUID 0x2AA0 before subscribing to notifications.
  • Reject payloads with length other than 4 bytes before parsing.
  • Decode as signed integers and convert units at presentation time only.
  • Store timestamped X/Y values for debugging and regression analysis.

FAQ

What is Bluetooth characteristic 0x2AA0?

It is Magnetic Flux Density 2D, used to report signed X-axis and Y-axis magnetic flux values.

How many bytes are in the payload?

4 bytes total: 2 bytes for X-axis and 2 bytes for Y-axis.

How do I decode negative values?

Use signed 16-bit little-endian parsing; two's complement handles negative values.

Does decimal exponent 0 require scaling?

No. The decoded integer already represents gauss directly.

How do I convert gauss to microtesla?

Multiply gauss by 100.
